git config --global user.name blabla - задати імя

git config --global user.email blabal@bla - задати адресу

git init - Ініціалізувати порожній локальний Git-репозиторій

git status - Перевірити стан

git add - Додати файл

git commit - Покласти зміни в локальний репозиторій

git log - Переглянути історію commit

git checkout - Переключитись між гілками

git branch - Показати лише локальні гілки

git push - Покласти зміни з локального у віддалений репозиторій

git pull - Оновити дані (забрати зміни) з віддаленого репозиторію по поточній гілці

git merge - Злити одну чи кілька гілок в основну

git clone - Клонувати репозиторій

.gitignore - Git ігноруватиме зміни цього файлу при виконанні команди git add і не включатиме його в коміти

git revert - Вона створює новий коміт, який вносить зміни, протилежні до зазначеного комміту, по суті скасовуючи його.

git reset - команда, що використовується для скасування локальних змін у репозиторії

git rebase - Автоматизована команда cherry-pick. Він виконує ту ж роботу, але для ланцюжка коммітів, тим самим ніби переносячи гілку на нове місце

git cherry-pick - бере зміни, що вносяться одним коммітом, і намагається повторно застосувати їх у вигляді нового комміту в поточній гілці

git stash - використовується для тимчасового збереження всіх незафіксованих змін для очищення робочого каталогу без необхідності фіксувати незавершену роботу в поточній гілці.
